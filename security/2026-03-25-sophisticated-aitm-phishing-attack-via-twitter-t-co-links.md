# Sophisticated AITM Phishing Attack via Twitter t.co Links

## Summary
This bookmarked tweet links to a Sublime Security article detailing a sophisticated Adversary-in-the-Middle (AITM) credential phishing attack. The attack uses X/Twitter's t.co link shortener to hide a malicious payload delivered via an email impersonating secure message notifications. It describes how the attackers employed brand confusion and urgency to trick users into divulging credentials, which were then harvested while also being passed to a legitimate OAuth server.

## Key Insights
- t.co links, while serving various purposes for Twitter (tracking, character count management, blocklisting), can be exploited by attackers to mask malicious URLs.
- Sophisticated phishing attacks, like AITM, can harvest credentials by acting as a proxy between the user and a legitimate authentication service, making detection harder for the end-user.
- Attackers use social engineering tactics such as brand confusion (e.g., DocuSign, ShareFile, Adobe, Microsoft), lookalike domains, and manufactured financial urgency to increase the likelihood of success.
- On mobile devices, it is particularly difficult for users to verify the true destination of t.co links, increasing vulnerability to deceptive links on platforms like X.

## Content Angles
- Could be used to create educational content or training materials for employees on how to identify and avoid sophisticated phishing attempts, especially those involving shortened URLs and brand impersonation.
- Could inform the development of security tools or browser extensions designed to unmask shortened URLs and analyze their true destination for potential threats before clicking.
